## 🤔 정규화란?
직역하면 '관계 정상화'라는 뜻으로,
데이터를 일정한 기준에 따르는 '정규적인' 상태로 바꾸거나,
비정상적인 형태를 정상으로 되돌리는 과정을 말한다.

--- 
## 📌 데이터베이스 정규화
데이터베이스 설계 과정에서 이상현상(anomaly)이 있는 릴레이션을 분해하여,
데이터 구조를 조직화하고 최적화하는 과정을 말한다.

--- 
## 📌 정규화의 목적
- **데이터 중복 최소화**
- 데이터 삽입, 수정, 삭제 시 발생할 수 있는 **이상 현상 방지**
- **데이터 무결성 유지**
- 관계의 명확화
- 효율성 향상


### 💡 이상 현상
정규화를 거치지 않은 데이터에서 발생할 수 있는 현상을 말한다.
- **삽입 이상 (Insertion Anomaly)** : 데이터 삽입 시 의도와 다른 값들도 삽입
- **삭제 이상 (Delete Anomaly)** : 데이터 삭제 시 의도와 다른 값들도 연쇄 삭제
- **갱신 이상 (Update Anomaly)** : 속성값 갱신 시 일부 튜플만 갱신되어 모순 발생


### 💡 무결성
데이터의 *정확성*, *일관성*, *유효성*이 유지되는 것
- **정확성** : 중복이나 누락이 없는 정확한 상태
- **일관성** : 원인과 결과의 의미가 연속적으로 보장되어 변하지 않는 상태
- **유효성** : 사용자로부터 값을 입력받을 때 유효한 값만 입력되는 상태

- 종류
  - 개체 무결성(Entity integrity)
  - 참조 무결성(Referential integrity)
  - 도메인 무결성(Domain integrity)
  - Null 무결성(Null integrity)
  - 고유 무결성 (Unique integrity)
  - 키 무결성 (Key integrity)
  - 관계 무결성 (Relationship integrity)

--- 
## 📌 정규화 단계
각 단계의 테이블은 **"정규형(Normal Form)"** 이라고 불리는 규칙을 따른다.

- **정규화** : 정리정돈 **과정** - 방 안에 있는 물건들을 정리한다.
- **정규형** : 정리정돈 **규칙** - 박스에는 하나의 물건만 넣는다.

--- 
## 1️⃣ 제 1정규형 (1NF, First Normal Form)
속성 값이 **원자적(Atomic)** 이어야 한다.
= 하나의 셀에 여러 값을 저장할 수 없다는 뜻

<img src="https://github.com/user-attachments/assets/2da2314d-a362-4fe9-98a1-6baac7ead58f" alt="제 1정규형 적용 전" width="700"/>

###### <제 1정규형 적용 전>

위의 테이블에서는 운영체제 과목을 듣는 수강자를 한번에 저장하고 있다.
하나의 컬럼에 다중 값이 저장된 상태이다.

### 💥 갱신 이상 발생

**예시 1) 일부만 값이 변경된다.**

> 운영체제 과목을 수강하는 수강자 중 홍길동이 개명해서 ​황길동으로 바꾸게 된다면,
> 자료구조 과목과 운영체제 과목의 모든 행에서 홍길동을 찾아서 수정해야 한다.
> 그러나 아래의 쿼리를 실행해보면 자료구조 수강자 홍길동만 황길동으로 변경된 것을 확인할 수 있다.
> 자료구조 수강자 홍길동은 여전히 남아있다. 속성값 갱신 시 일부 튜플만 갱신되어 모순 발생한 경우로,
> 이로 인해 **데이터 일관성** 이 깨진 것을 확인할 수 있다.

``` sql
UPDATE `과목테이블`
SET `수강자` = '황길동'
WHERE `수강자` = '홍길동';
```

<img src="https://github.com/user-attachments/assets/1bf45c5c-c82b-4d7c-9fdb-6fd3f6f36d56" alt="제 1정규화 - 갱신 이상 1" width="700"/>


**예시 2) 하나의 값만 변경하고 싶어도 전체 값이 함께 변경된다.**

> '홍길동'을 직접 찾지 않고 LIKE로 찾아서 업데이트하면 함께 변경되지 않을까 싶겠지만,
> 이런 경우는 해당 컬럼의 전체 값이 변경된다.
> 속성값 갱신 시 의도하지 않은 값이 갱신되어 모순이 발생한 경우이다.

``` sql
UPDATE `과목테이블`
SET `수강자` = '황길동'
WHERE `수강자` LIKE '%홍길동%';
```

<img src="https://github.com/user-attachments/assets/1d7994e6-d1ea-4c8d-bc7e-4f930cb32f94" alt="제 1정규화 - 갱신 이상 2" width="700"/>

</br>

### 💥 삭제 이상 발생

**예시 1) 일부 값만 삭제된다.**

> 홍길동이 자퇴를 해서 더이상 과목을 수강하지 않는다. 이 경우 홍길동의 모든 데이터를 삭제하려고 했지만,
> 다중 값이 저장된 컬럼에서는 삭제가 되지 않는 것을 확인할 수 있다.

``` sql
DELETE FROM `과목테이블`  WHERE `수강자` = '홍길동';
```

<img src="https://github.com/user-attachments/assets/6ddf0caf-ae10-4b66-bacd-f0d8b8501b1c" alt="제 1정규화 - 삭제 이상 1" width="700"/>


**예시 2) 원하지 않는 데이터가 함께 삭제된다.**

> 홍길동이 삭제되면 운영체제를 같이 듣는 김철수도 함께 삭제된다.

``` sql
DELETE FROM `과목테이블`  WHERE `수강자` LIKE '%홍길동%';
```

<img src="https://github.com/user-attachments/assets/115a5ce5-d740-4849-b9be-58bf4e91845f" alt="제 1정규화 - 삭제 이상 2" width="700"/>

> 또한 공통적으로
> 해당 테이블은 과목 ID가 키값인데, 수강자 컬럼의 값만 삭제되는 것이 아닌 튜플 전체가 삭제되어버린다.

## 2️⃣ 제 2정규형 (2NF, Second Normal Form)
1NF 만족, **부분 함수 종속성을 제거**해야 한다.
기본 키의 부분집합에 의존하는 컬럼들로만 구성되어야 한다.

<img src="https://github.com/user-attachments/assets/51707536-3da8-4e81-8212-da0ca2b21984" alt="제 2정규형 적용 전" width="700"/>

###### <제 2정규형 적용 전>

위의 테이블에서는 과목 ID에 대한 종속성을 가지는 데이터와, 학생 ID에 대한 종속성을 가지는 데이터를
하나의 테이블에 가지고 있다.


### 💥삽입 이상 발생
> 새로운 과목이 추가될 때 불필요하게 해당 과목을 수강하는 학생 정보까지 함께 입력해야 한다.
> 원하는 정보만 추가할 수 없는 것이다.

``` sql
INSERT INTO `과목테이블`(`과목ID`, `과목명`, `강의실`, `수강자`, `나이`, `학생 ID`)
VALUES (2, '운영체제', '301호', '홍길동', 21, '101');
INSERT INTO `과목테이블`(`과목ID`, `과목명`, `강의실`, `수강자`, `나이`, `학생 ID`)
VALUES (2, '운영체제', '301호', '김철수', 22, '102');
```


### 💥갱신 이상 발생

> 일부만 값이 변경된다.


### 💥삭제 이상 발생

> 원하지 않는 데이터가 함께 삭제된다.
> 수강자 정보가 삭제되어도 과목에 대한 정보는 남아있어야 한다.

## 3️⃣ 제 3정규형 (3NF, Third Normal Form)
1NF 2NF 만족, **이행적 함수 종속성을 제거**해야 한다.
비주요 속성들이 다른 비주요 속성에 의존할 때, 이행적 종속이 발생한다.
이런 경우 해결을 위해 테이블을 분리해야 한다.


### 🤔 이행적 함수 종속?
**A → B** (종속) 이고 **B → C** (종속) 이라면 **A → C** (종속) 인 형태

<img src="https://github.com/user-attachments/assets/9bec6797-6769-49c1-8caa-bb6c9fd79337" alt="제 3정규형 적용 전" width="700"/>

###### <제 3정규형 적용 전>

공학인증 여부는 공학인증 번호에 종속되는 개념이다.
과목 정보(주요 속성)이 아닌 비주요 속성에 의존하고 있다.

​
### 💥 삽입 이상 발생
> 일부 데이터만 추가될 때, 종속되지 않은 데이터가 null로 추가된다.


### 💥 갱신 이상 발생
> 상단의 데이터가 변경되면, 중복 데이터로 인해 하위 여러 행을 수정해야 한다.


### 💥 삭제 이상 발생
> 원하지 않는 데이터가 함께 삭제된다.

</br>

###### <제 3정규화 테이블>

< 과목 ID → 공학인증 번호 → 공학인증 여부 >

## 📌 정규화의 장단점
정규화를 단계별로 진행하면서 장점에 대해서는 충분히 이해했다.
그렇다면 단점은 무엇이 있고, 어떤 경우에 정규화를 진행하지 않는걸까?


- 장점
  - 올바른 데이터를 얻을 수 있다.
  - 불필요한 쿼리 제거로 성능이 향상된다.
- 단점
  - 테이블 분해로 인해 테이블 간의 연산(JOIN)이 많아진다. = 응답 시간이 느려질 수 있다.

---
## 📌 정규화 대상

CRUD가 많이 일어나는 거래 시스템 등을 대상으로 한다.

---
### 🤔 반정규화?
정규화된 시스템을 성능 향상 및 개발과 운영의 단순화를 위해 역으로 정규화를 수행하는 것을 말한다.
일반적으로 JOIN이 많이 사용되고, 많은 범위의 조회를 자주 처리할 때 부분적으로 반정규화를 진행한다.

  - 디자이너 정보
  - 주소 정보
  - 이미지 URL


- 포트폴리오 테이블
  - 디자이너 아이디 (FK)
  - 포트폴리오 내용
  - 동영상 URL


- 포트폴리오 이미지 테이블
  - 포트폴리오 아이디 (FK)
  - 이미지 URL

- 설명
  한 명의 디자이너는 3개의 포트폴리오를 가질 수 있다.
  하나의 포트폴리오에는 1개의 동영상과 3개의 이미지를 업로드할 수 있다.
